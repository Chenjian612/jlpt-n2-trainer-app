# JLPT N2 Trainer App

一个面向 JLPT N2 备考的 Expo / React Native 训练应用。当前版本已经覆盖首页推荐、刷题、记忆包、读解、听力、错题回收和本地进度记录，适合日常碎片化练习与阶段性复盘。

当前这版已经完成一轮主界面产品化升级，并把薄弱点系统从文法/词汇刷题扩展到了读解与听力：首页、刷题、记忆包、读解、听力等主路径在视觉层级、反馈结构和移动端可读性上都做了统一。

## 当前能力

- 首页根据当天完成情况、错题积压和薄弱点信号，动态推荐下一轮训练。
- 支持 8 个训练模式：
  - 文法闯关
  - 文法记忆包
  - 词汇刷题
  - 词汇记忆包
  - 读解实战
  - 听力要点拆解
  - 文法错题回收
  - 词汇错题回收
- 首页、刷题、记忆包、读解、听力页已完成一轮统一视觉升级，训练主路径的标题、反馈和行动按钮保持同一套语气。
- 文法/词汇刷题结果会自动写入本地进度；答错的题会自动进入回收队列。
- 错题回收支持优先级排序，优先处理高频错误、待首次回收和该复习的题。
- 读解与听力会把题级错误沉淀成 `weaknessSignals`，进入首页“薄弱点摘要”和“今天怎么攻克”。
- 听力模式已接入官方公开示例音频改编材料，支持播放、重播、倍速和听后复盘。
- 读解、刷题、记忆包、错题回收都提供结果页与下一步建议。

## 最近更新

2026-03-13 这轮更新重点是把“薄弱点系统”从刷题扩展到四大主训练中的读解和听力：

- 新增独立的 `weaknessSignals` 持久化层，用来记录读解和听力的题级弱点，不与文法/词汇错题回收队列混用。
- 首页“薄弱点摘要”现在会同时聚合：
  - 文法/词汇刷题错题
  - 读解弱点信号
  - 听力弱点信号
- 读解新增错误类型：
  - 读解证据定位
  - 读解主旨判断
  - 读解干扰项排除
- 听力新增错误类型：
  - 听力转折漏听
  - 听力信息追踪
  - 听力最终结论
  - 听力主任务判断
- 首页攻克计划会根据弱点来源自动回推到对应模式，不再只把用户送去文法/词汇错题回收。

## 目录结构

- `src/app`：应用根节点、Provider、导航。
- `src/features`：首页、模式详情、刷题、记忆包、读解、听力、错题回收等功能页。
- `src/domain`：领域模型与进度/推荐/教练服务。
- `src/data`：seed 数据、题库、音频材料、仓储与存储封装。
- `src/components`：通用 UI 组件。
- `src/theme`：颜色、字体、圆角、阴影等设计 token。

## 本地运行

安装依赖：

```bash
npm install
```

启动开发服务：

```bash
npm run start
```

常用目标：

```bash
npm run web
npm run android
npm run ios
```

如果要用 Expo Go 做远程预览，请使用 Expo 默认的 Metro 端口 `8081` 并配合 `tunnel` 模式。

## 数据与状态

- 训练进度保存在本地 `AsyncStorage`。
- 存储键为 `jlpt-n2-trainer-state-v1`。
- 记录内容包括：
  - 每日训练轮次
  - 每个模式的完成情况
  - 文法/词汇错题列表与最近误选
  - 错题回收状态
  - 读解/听力 `weaknessSignals`
  - 周目标与连续天数相关统计

## 已做验证

当前版本已经做过一轮浏览器级与服务层组合验证，确认通过的内容包括：

- 首页推荐与模式进入
- 文法刷题 -> 连续答题 -> 结果页展示 -> 返回首页建议
- 文法刷题 -> 生成错题 -> 错题回收 -> 返回首页
- 文法记忆包进入与翻卡
- 读解整轮完成后会影响首页“薄弱点摘要”
- 听力首题可完成“播放 -> 选择 -> 提交 -> 进入讲解态”
- 教练服务层可正确聚合 reading / listening 的 weakness signal，并在纠正后清除对应弱点
