# Codex 任务指令包

> 使用方法：按顺序逐一发给 Codex，跑完一个再发下一个。
> 每次跑完让 Codex 执行 `npm test` 验证格式，再继续下一轮。
> 如果 Codex 卡在 shell 命令不动，发「停止所有命令，直接输出 JSON 文本给我」。

---

## 注意事项

- 每次只做一个文件，不要同时改多个文件
- 每轮生成完把结果发给 Claude 检查，确认无误再追加到文件
- 如果 Codex 卡死在 shell 命令，让他跳过读文件步骤，直接输出 JSON 文本

## 任务一：文法刷题补充

> 当前：20 题（grammar-001 至 grammar-020）
> 目标：500+ 题
> 每次生成 10 题，source 依次改为「题组 B」「题组 C」「题组 D」……

```
【角色】
你是一位 JLPT N2 备考教材编写专家，熟悉日语能力考试的出题规律和考点分布。

【准备】
请先读取 src/data/seed/drill_questions.json，确认现有结构。
当前共 40 题，grammar_drill 最大编号为 grammar-020，本次从 grammar-021 开始。

【任务】
向 src/data/seed/drill_questions.json 末尾追加 10 道文法刷题，不要修改文件中已有的任何条目。

【字段要求】
每条严格遵守以下结构：
{
  "id": "grammar-XXX",
  "modeId": "grammar_drill",
  "prompt": "日语句子，空白处用 ___ 表示",
  "choices": ["选项A", "选项B", "选项C", "选项D"],
  "answer": 0,
  "explanation": "中文解析，说明为何选此项、其他三项为何错",
  "choiceInsights": [
    "选项A的对错原因",
    "选项B的对错原因",
    "选项C的对错原因",
    "选项D的对错原因"
  ],
  "reviewNote": "一句话复习要点",
  "tags": ["文法", "具体文法点名称", "考点类型"],
  "source": "N2 文法迷你题组 B"
}

【内容要求】
- 难度对应真实 JLPT N2 水平
- 每题考查不同的 N2 文法点，不与现有 grammar-001 至 grammar-020 重复
- 四个选项须为容易混淆的近义文法，干扰项要有真实迷惑性
- explanation 和 choiceInsights 用中文撰写，解析清晰专业
- tags 参考现有题目风格，用中文或日文标注核心考点，1-3 个
- 10 题覆盖：接续用法、转折让步、条件限制、强调结论、并列递进等
```

## 任务三：文法学习项补充

> 当前：15 项（grammar-study-1 至 grammar-study-15）
> 目标：100+ 项
> 每次生成 15 项，跑 6 次即可到 100 项

```
【角色】
你是一位 JLPT N2 备考教材编写专家，熟悉日语能力考试的出题规律和考点分布。

【准备】
请先读取 src/data/seed/grammar_study_items.json，确认现有结构。
当前共 15 项，最大编号为 grammar-study-15，本次从 grammar-study-16 开始。

【任务】
向 src/data/seed/grammar_study_items.json 末尾追加 15 个文法学习项，不要修改文件中已有的任何条目。

【字段要求】
每条严格遵守以下结构：
{
  "id": "grammar-study-XX",
  "modeId": "grammar_study",
  "term": "文法形式",
  "coreMeaning": "一句话说明核心含义（中文）",
  "keyUsage": "接续方式和适用场景（中文）",
  "confusingPair": "与最常混淆的文法点对比，说明区别（中文）",
  "example": "一个自然的日语例句",
  "memoryHook": "一句记忆口诀或联想方法（中文，15 字以内）",
  "reviewPrompt": "一个让学习者自我检验的反问句（中文）"
}

【内容要求】
- 难度对应真实 JLPT N2 水平
- 不与现有 15 项重复，现有项包含：
  にしたがって、わけではない、ことになっている、ものの、末に、
  にすぎない、ことから、かねない、一方だ、どころか、ばかりに、
  に応じて、うえで、ことなく、とは限らない
- confusingPair 必须真实存在容易混淆的文法，不要编造对比
- example 例句要自然，符合 N2 书面语风格
- memoryHook 简洁有趣，15 字以内
```

## 任务四：读解素材补充

> 当前：6 篇（reading-001 至 reading-006）
> 目标：15+ 篇
> 每次生成 2 篇，跑 5 次即可到 16 篇

```
【角色】
你是一位 JLPT N2 备考教材编写专家，熟悉日语能力考试的出题规律和考点分布。

【准备】
请先读取 src/data/seed/reading_passages.json，确认现有结构。
当前共 6 篇，最大编号为 reading-006，本次从 reading-007 开始。

【任务】
向 src/data/seed/reading_passages.json 末尾追加 2 篇读解文章，不要修改文件中已有的任何条目。

【字段要求】
每篇严格遵守以下结构：
{
  "id": "reading-XXX",
  "modeId": "reading_drill",
  "title": "文章标题（中文）",
  "source": "原创读解素材（参考 JLPT N2 说明文风格）",
  "lead": "导读句，简要说明文章背景和阅读重点（中文，2-3 句）",
  "guidance": "阅读指导，告诉学习者如何拆解文章结构（中文，1-2 句）",
  "paragraphs": [
    "第一段（日语）",
    "第二段（日语）",
    "第三段（日语）",
    "第四段（日语）"
  ],
  "questions": [
    {
      "id": "reading-XXX-q1",
      "prompt": "题干（日语）",
      "choices": ["选项A（日语）", "选项B（日语）", "选项C（日语）", "选项D（日语）"],
      "answer": 0,
      "explanation": "中文解析，说明为何选此项、其他三项为何错",
      "choiceInsights": [
        "选项A的对错原因（中文）",
        "选项B的对错原因（中文）",
        "选项C的对错原因（中文）",
        "选项D的对错原因（中文）"
      ],
      "evidence": "证据位置：第 X 段，引用原文关键句",
      "reviewNote": "一句话阅读方法提示（中文）",
      "tags": ["考点类型"]
    }
  ]
}

【内容要求】
- 正文每段 150-200 日语字，共 4 段，总字数 600-800 字
- 难度对应 JLPT N2，使用书面语，适当出现 N2 文法和词汇
- 2 篇文章题材不同，建议一篇说明文、一篇议论文
- 每篇 4 道题，考点分布：主旨题 1 道、细节题 1 道、推断题 1 道、转折或结果题 1 道
- tags 从以下值中选 1-2 个：主旨、细节题、推断题、转折判断、结果题
- 干扰项要有真实迷惑性，不能一眼排除
- evidence 必须引用正文原句，不能空着
```
