## JSON 字段规范

每道题是 `drill_questions.json` 数组里的一个对象，字段如下：

```jsonc
{
  "id": "grammar-151",           // 严格递增，三位数补零（grammar-001 / vocab-001）
  "modeId": "grammar_drill",     // 文法题固定填 "grammar_drill"，词汇题填 "vocab_drill"
  "prompt": "〜___〜",           // 题干，空格用 ___ 表示，必须是完整日语句子
  "choices": [                   // 恰好 4 个选项，正确答案不固定放哪一位
    "选项A",
    "选项B",
    "选项C",
    "选项D"
  ],
  "answer": 0,                   // 正确选项的 0-based 索引（0=A, 1=B, 2=C, 3=D）
  "explanation": "...",          // 中文，1~3句，解释为什么这个句型/词汇是正确的
  "choiceInsights": [            // 恰好 4 个字符串，顺序与 choices 一一对应
    "为什么A对或错",              // 每条 1~2句，中文，重点说清楚该选项的语义和为何不适合
    "为什么B对或错",
    "为什么C对或错",
    "为什么D对或错"
  ],
  "reviewNote": "...",           // 中文，1句话，提炼出做这类题的通用思路或记忆技巧
  "tags": [                      // 第一个 tag 固定是 "文法" 或 "词汇"，后面加具体考点
    "文法",
    "句型名称",
    "语义分类"
  ],
  "source": "N2 文法迷你题组 X"  // 自由填写，用于标注来源批次
}
```

## 内容质量要求

### 文法题

1. **考点明确**：每题聚焦一个 N2 文法句型，四个选项之间必须有混淆性（同类句型、近义表达）
2. **干扰项有效**：不能放明显错误的选项，每个干扰项都要"看起来可能对"
3. **解释要讲透**：`explanation` 说清楚正确句型的语义；`choiceInsights` 逐一点出每个选项为什么对/错
4. **reviewNote 要可复用**：不是"这题考XX"，而是"遇到XX情况时，做题思路是YY"
5. **正确答案位置随机**：四道题里正确答案不要总在 index 0，均匀分布在 0~3

### 词汇题

1. 考查 N2 核心词汇的意思辨析、用法、搭配
2. 干扰项选近义词或同类词，不选明显无关词
3. 句子语境要能帮助推断词义，不能出"没有语境"的纯记忆题

### 通用禁止项

- 禁止重复已有题目的考点句型（先检查现有题目的 tags）
- 禁止出现语法错误的日语句子
- 禁止 `choiceInsights` 里解释顺序与 `choices` 顺序不一致
- 禁止 `answer` 填错（务必确认 0-based 索引和 choices 数组的对应关系）
- 禁止把内容写进 `.ts` 文件，所有题目只能加在 JSON 文件里

## 执行指令（直接给 AI 的任务描述模板）

### 添加文法题

```
请向 src/data/seed/drill_questions.json 中 grammar-200 之后、vocab-041 之前的位置
追加 N 道 N2 文法刷题题目，ID 从 grammar-XXX 开始递增。

要求：
- 严格遵守 CONTENT-SPEC.md 的 JSON 字段规范
- 不重复已覆盖考点（见 CONTENT-SPEC.md 已覆盖列表）
- 每题正确答案随机分布在 0~3
- explanation、choiceInsights、reviewNote 全部用中文
- 追加到 JSON 数组里，不新建文件，不修改已有题目
```

### 添加词汇题

```
请向 src/data/seed/drill_questions.json 末尾
追加 N 道 N2 词汇刷题题目，ID 从 vocab-XXX 开始递增。

要求同上，modeId 改为 "vocab_drill"，tags 第一个改为 "词汇"。
```
